10 个配色工作流

按场景选 skill，复制命令后补充项目背景。

选一个 /xxd-* 命令，补充场景、受众、已有色值或截图。

01
定方向

xxd-color-brief

把模糊审美词变成可执行约束。

痛点：客户只说感觉 产出：配色简报 亮点：先对齐判断标准

价值：先把“感觉”变成共识。

GitHub 目录
价值

先统一判断语言，再选色。

痛点

只有“高级、东方、年轻”等模糊词。

适合

项目初期、客户反馈虚、只有参考图。

输入

`/xxd-color-brief` + 载体、受众、参考图、避开气质。

产出

色彩约束、起始色和下一步 skill。

示例触发 /xxd-color-brief 新中式茶饮品牌配色方向
02
组色板

xxd-palette-builder

从太多选择里筛出能用的色板。

痛点：色板好看但难选 产出：主辅点缀方案 亮点：每个色都有角色

价值：把选择困难变成少量方案。

GitHub 目录
价值

从 742 色里收敛出可用色板。

痛点

颜色太多，不知道哪组能用。

适合

已有锚点色、HEX、情绪词或场景。

输入

`/xxd-palette-builder` + 锚点色、用途、稳妥/醒目。

产出

主色、辅色、背景、强调和比例。

示例触发 /xxd-palette-builder 月白 网站首页色板
03
放版面

xxd-palette-applier

把漂亮色板真正落到版面里。

痛点：不知道放哪里 产出：版面用色地图 亮点：主次和面积更清楚

价值：让用户先看到重点。

GitHub 目录
价值

把色板分配到背景、内容和行动位。

痛点

色板好看，落版后主次混乱。

适合

海报、网页、PPT、包装、封面或 UI。

输入

`/xxd-palette-applier` + 色板、载体、信息层级。

产出

角色、面积比例和错误用法。

示例触发 /xxd-palette-applier 课程封面首屏
04
交 token

xxd-ui-token

把传统色变成开发能接的 UI token。

痛点：HEX 难交付 产出：UI token 亮点：浅深色模式可接入

价值：设计和开发共用命名。

GitHub 目录
价值

把色板转成可维护的团队资产。

痛点

只交 HEX，状态和暗色模式难维护。

适合

网站、App、后台、Figma variables、Tailwind。

输入

`/xxd-ui-token` + 产品类型、模式、输出格式。

产出

CSS、Tailwind 或 Figma token。

示例触发 /xxd-ui-token 官网浅色和暗色 token
05
查可读

xxd-accessible-color

让好看的传统色也能读得清。

痛点：漂亮但不清楚 产出：可读性修复 亮点：给出通过和替代色

价值：好看也要读得清。

GitHub 目录
价值

检查文字、按钮、状态和图表可读性。

痛点

传统色雅致，但压文字可能看不清。

适合

正文、按钮、焦点环、状态色、图表。

输入

`/xxd-accessible-color` + 前景色、背景色、用途。

产出

对比度、通过判断和替代色。

示例触发 /xxd-accessible-color 检查按钮可读性
06
定品牌

xxd-brand-system

把一次配色沉淀成长期品牌规则。

痛点：品牌色会漂移 产出：品牌用色规范 亮点：明确禁用组合

价值：少为品牌色反复争论。

GitHub 目录
价值

把一次配色沉淀成长期规则。

痛点

每次物料都重配，品牌识别漂移。

适合

品牌升级、官网、社媒、包装、产品 UI。

输入

`/xxd-brand-system` + 行业、价格带、竞品、渠道。

产出

锚点色、支撑色、比例和禁用组合。

示例触发 /xxd-brand-system 文创香氛品牌色规范
07
做图表

xxd-data-viz

按数据含义生成传统色图表方案。

痛点：图表颜色难分 产出：图表色数组 亮点：分类顺序发散分开处理

价值：先读懂数据，再感受色彩。

GitHub 目录
价值

按数据关系分配颜色。

痛点

分类不清、顺序不明、发散无中点。

适合

柱状图、折线图、地图、热力图、看板。

输入

`/xxd-data-viz` + 图表类型、系列数量、数据关系。

产出

分类色、色带、高亮色或状态色。

示例触发 /xxd-data-viz 7 类图表色
08
修旧稿

xxd-existing-design-audit

把旧稿和散乱 HEX 修成传统色系统。

痛点：旧稿颜色混乱 产出：保留替换清单 亮点：不盲目推翻旧设计

价值：保留有效旧资产。

GitHub 目录
价值

盘点旧色，给出保留/替换边界。

痛点

旧稿里相近色、重复色、临时色太多。

适合

已有截图、CSS variables、Figma styles。

输入

`/xxd-existing-design-audit` + HEX 清单、截图、当前角色。

产出

映射、保留、合并、替换、移除建议。

示例触发 /xxd-existing-design-audit 审计 CSS 颜色
09
做系列

xxd-content-series

让系列内容既统一又不疲劳。

痛点：系列不成体系 产出：轮换规则 亮点：固定层和变化层分开

价值：系列稳定，不用每期重配。

GitHub 目录
价值

把系列内容拆成固定层和变化层。

痛点

每期重配会乱，完全固定会腻。

适合

小红书、公众号、视频封面、课程栏目。

输入

`/xxd-content-series` + 平台、栏目、节奏、模板。

产出

固定层、变量层、栏目色和轮换规则。

示例触发 /xxd-content-series 小红书 5 栏目封面色
10
进生产

xxd-print-packaging

把屏幕色推进到包装和印刷生产。

痛点：屏幕色不等于印刷 产出：打样检查清单 亮点：把材质和工艺纳入判断

价值：提前看到印刷风险。

GitHub 目录
价值

把屏幕配色翻译到生产约束。

痛点

HEX 不等于印刷效果。

适合

茶、香、书籍、礼盒、文创、标签。

输入

`/xxd-print-packaging` + 品类、材质、工艺、场景。

产出

面板用色、系列规则、打样检查清单。

示例触发 /xxd-print-packaging 茶叶礼盒打样清单
