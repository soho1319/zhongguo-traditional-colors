# 中国传统色配色用途说明

这份说明由 `scripts/build-harmony-use-cases.mjs` 自动生成，和 `docs/chinese-color-harmony-use-cases.csv` 使用同一套用途数据。

CSV 覆盖每个中国传统色与每一种配色逻辑：同类、邻近、互补、分裂互补、三角、四角、冷暖、明色、暗色、灰调、中性。每行都包含适合方向、典型场景、设计师用法、普通人理解、面积建议、风险提醒、逐行判断依据、关系色分工、具体使用动作、不适合场景、审稿检查问题和当前色落地建议。

- 颜色数量：742
- 配色逻辑：11
- 表格行数：8162

## 配色逻辑用途矩阵

| 配色逻辑 | 适合方向 | 典型场景 | 面积建议 | 风险提醒 |
| --- | --- | --- | --- | --- |
| 同类 / 统一 | 保持同一种气质，让画面稳定、安静、低风险。 | 品牌延展、资料页背景、系列封面、文化内容、长文配图、低干扰界面模块。 | 主色 60-80%，同类关系色 15-35%，强调色控制在 5% 以内。 | 容易过于平，建议用字号、留白、明暗或材质建立层级；如果主色本身很浅，必须另选深色保证文字可读。 |
| 邻近 / 柔和 | 在统一中加入轻微转折，让画面有变化但不跳脱。 | 渐变背景、插画分层、节气海报、内容系列、温和品牌视觉、课程封面。 | 主色 50-70%，邻近色 25-45%，强对比色不超过 5%。 | 相邻色过多会显得糊，建议限制在 2-3 个，并用明暗差拉开信息层级。 |
| 互补 / 强调 | 制造一眼可见的焦点，用于让按钮、标题、数字或行动入口被看见。 | 海报主标题、CTA 按钮、价格数字、重点标签、活动主视觉、封面视觉锤。 | 主色 70-85%，互补色 5-15%，中性色或留白 10-25%。 | 大面积互补容易刺眼或显廉价；文字放在互补色上必须检查对比度，不要只凭感觉。 |
| 分裂互补 / 稳对比 | 保留对比张力，但比直接互补更柔和、更容易进入现代版面。 | 网页首屏、品牌活动页、封面辅助图形、内容重点、包装辅助色、界面状态色。 | 主色 60-75%，分裂互补色合计 10-25%，结构中性色 15-30%。 | 两个关系色同时使用时要有主次；不要把它们都做成按钮色或标题色。 |
| 三角 / 系列感 | 建立三种方向的色彩节奏，适合多张内容或多模块系统。 | 系列海报、课程章节、社媒栏目、图表分类、活动视觉系统、内容矩阵。 | 单张画面中主色 50-70%，其余三角色择 1-2 个使用；系列物料中可以轮换主辅。 | 同屏使用过多会变花；最好先定义每个颜色负责的内容类型。 |
| 四角 / 多层级 | 给复杂信息提供更多层级选择，用于大型专题或多状态系统。 | 大型专题页、复杂活动物料、产品矩阵、数据看板、包装系列、多状态 UI。 | 主色 45-65%，第二色 15-25%，第三色 5-12%，第四色只做状态或小面积点缀。 | 最容易失控。需要先建立用途规则，否则看起来像随机拼色。 |
| 冷暖 / 情绪差 | 用冷暖变化制造情绪转折，让主体、信息和氛围分开。 | 空间海报、节气/天气内容、文化展览、品牌情绪版、产品上新、故事型长图。 | 按情绪选择主色 60-75%，另一侧色温 10-25%，中性色 10-25%。 | 冷暖同时强饱和会显吵；要用明度和饱和度控制距离。 |
| 明色 / 浅背景 | 把主色变轻，生成背景、留白、柔和提示和低压阅读环境。 | 网页背景、卡片底色、PPT 页面、长文阅读、轻提示、资料库界面。 | 明色可占 70-90%，主色或深色文字 8-20%，点缀色 2-8%。 | 浅色很容易不够可读，尤其在投影、低亮度屏幕和打印时更明显。 |
| 暗色 / 深文字 | 从主色关系里找更稳的文字、边界、深色底和压重点。 | 标题文字、正文色、深色模式、边框线、封面压重、海报底部信息。 | 暗色文字/边界 10-25%；若做深底，暗色可占 60-80%，亮色文字需要复核对比度。 | 暗色过多会沉闷；深底上的彩色文字要避免发灰或糊在一起。 |
| 灰调 / 降噪 | 降低饱和度，让颜色更耐看、更适合长时间阅读和信息密集场景。 | 后台界面、资料页、长文、表格、品牌规范、商务提案、低调包装。 | 灰调 50-80%，主色 10-25%，强调色 3-8%。 | 灰调过多会没有记忆点；需要保留一个更明确的主色或强调色。 |
| 中性 / 留白 | 为主色提供结构、留白、边界和呼吸感，让信息更容易被看懂。 | 网页骨架、排版留白、表格线、边框、背景、信息卡片、品牌基础色。 | 中性色 60-85%，主色 10-25%，强调色 2-8%。 | 中性色太接近主色时层级会不清；需要用明度差和字号建立秩序。 |

## 使用方式

1. 先在 CSV 中找到目标色名或编号。
2. 再按“配色逻辑”筛选同类、互补、灰调等方向。
3. 先读“逐行判断依据”，确认这一行为什么推荐这组关系色。
4. 再读“关系色分工”和“具体使用动作”，决定哪些色用于背景、标题、正文、按钮或点缀。
5. 最后用“不适合场景”和“审稿检查问题”反向检查，避免把配色关系机械套用。
